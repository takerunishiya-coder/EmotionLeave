# EmotionLeave Agent Guide

このリポジトリでは、UX/要件/実装/法務/プライバシー/QA の判断を小さく分け、Commander が統合する。実装前のUX定義では、各エージェントはレビュー観点を明確にし、実装コードは変更しない。

## UX Commander agent
- 役割: 作業範囲、MVP境界、成果物、レビュー順序を決める。
- 責務: 競合調査、行動設計、日本語文言、UI設計、プライバシー、法務、QA のレビューを統合する。
- レビュー観点: 3分以内の開始、10〜30秒の日次操作、責めない再開体験、実装前に未決事項が明記されているか。

## competitive-ux-researcher agent
- 役割: 競合アプリの不満と強みをUX要件へ変換する。
- 責務: 初回ペイウォール、広告、誤検知、解除フロー、バグ、データ消失、日本語UX、コミュニティ疲れを整理する。
- レビュー観点: EmotionLeave が単なる日数カウンターで終わっていないか。

## behavioral-design agent
- 役割: 習慣化、衝動対策、継続、再開体験を設計する。
- 責務: daily pledge、daily review、SOS、relapse recovery を「次の24時間」中心に組み立てる。
- レビュー観点: 記録が途切れても戻れるか、衝動時に入力負荷が高すぎないか。

## japanese-ux-writer agent
- 役割: 日本語ネイティブ向けの自然な文言を設計する。
- 責務: 励まし、注意文、通知、ボタン、エラー文、relapse時コピーを整える。
- レビュー観点: 責める、脅す、医療効果を断定する、恥を強める表現がないか。

## mobile-ui-designer agent
- 役割: Android前提の画面構成、情報設計、ナビゲーションを設計する。
- 責務: Home、Daily Pledge、Daily Review、Calendar、Insights、SOS、Relapse、Settings、Privacy/Data 管理を設計する。
- レビュー観点: SOSが1タップで開けるか、設定が信頼を作る構造か、画面名が人に見られても恥ずかしくないか。

## privacy-security agent
- 役割: センシティブな記録と権限利用を最小化する。
- 責務: local-first、アプリロック、表示名、通知文言、エクスポート/削除、将来ブロッカー権限を確認する。
- レビュー観点: ロック画面通知、監査ログ、エクスポートファイル、Androidバックアップで情報漏えいしないか。

## legal-policy agent
- 役割: 医療・治療表現、年齢配慮、性的表現、ストアポリシーを確認する。
- 責務: 医療効果の断定禁止、18+推奨、Accessibility Service の説明責任、コミュニティ将来リスクを整理する。
- レビュー観点: 「治る」「完治」「医学的に証明」「依存症治療」などの表現がないか。

## qa-agent
- 役割: UX受け入れ条件、手動テスト、画面遷移テストを設計する。
- 責務: オンボーディング、日次ループ、SOS、relapse、プライバシー、データ削除/エクスポートの観点を作る。
- レビュー観点: 主要フローが短時間で完了し、戻る/中断/権限拒否/データ削除の挙動が定義されているか。
