# User Stories

## Priority Model

EmotionLeaveの勝ち筋は、日数を数えることではなく、崩れそうな瞬間と崩れた直後を支えることにある。特に重視するユーザーストーリーは次の4つ。

1. 衝動が来た瞬間に、迷わずSOS画面へ行ける。
2. 記録が途切れても、責められずに再開できる。
3. 毎日10〜30秒でチェックインできる。
4. 人に見られても恥ずかしくない形で使える。

## US-01 初めて使うユーザー

- ユーザーストーリー: 禁欲や衝動コントロールを始めたいユーザーとして、私は難しい設定なしでアプリを始めたい。なぜなら、最初から面倒だと使う前に離脱してしまうから。
- 背景課題: 初回から課金、広告、重い権限、診断のような質問が出ると不信感と羞恥が生まれる。
- 体験イメージ: アプリを開き、3分以内に `開始日`, `目標`, `取り戻したいもの` を登録し、まず記録を始める。
- 対応画面: Splash, Onboarding, Goal Setup, First Daily Pledge, Home
- MVP: 入れる
- 優先度: P0
- 受け入れ条件:
  - 初回設定は3分以内でHomeへ到達できる。
  - 初回に課金、広告、評価依頼、アカウント作成、重い権限要求が出ない。
  - 開始日、目標、取り戻したいものを登録できる。
  - `禁欲に失敗しないため` ではなく、`生活を整えるため` の説明になっている。
  - 詳細な性的内容、実名、医療歴を聞かない。

## US-02 毎日続けたいユーザー

- ユーザーストーリー: 継続中のユーザーとして、私は毎日短時間で状態を確認したい。なぜなら、記録が重いと継続そのものが負担になるから。
- 背景課題: 記録が宿題化すると、アプリを開くこと自体が負担になる。
- 体験イメージ: 朝は `今日の一言宣言`、夜は `今日の振り返り` を10〜30秒で完了する。
- 対応画面: Home, Daily Pledge, Daily Review, Calendar, Insights
- MVP: 入れる
- 優先度: P0
- 受け入れ条件:
  - quick path は10〜30秒で完了できる。
  - 標準入力でも1分以内、詳細レビューでも2分以内に収まる。
  - Homeに現行記録、最長記録、累計成功時間、再開回数、今日のひとことが表示される。
  - 継続日数だけが突出して大きい設計になっていない。
  - missed check-inでも記録は途切れず、Homeに `今からでも大丈夫です` を表示する。

## US-03 衝動が来たユーザー

- ユーザーストーリー: 強い衝動が来たユーザーとして、私はすぐに気をそらす行動を取りたい。なぜなら、その瞬間に迷うとアプリを閉じてしまうから。
- 背景課題: 高衝動時は説明を読めず、選択肢が多いほど離脱する。
- 体験イメージ: HomeからワンタップでSOSへ入り、10秒、60秒、3分の行動を実行する。
- 対応画面: Home, SOS, Later Reflection, Daily Review, Relapse Log
- MVP: 入れる
- 優先度: P0 Critical
- 受け入れ条件:
  - Homeの第一表示範囲からSOSへ1タップで遷移できる。
  - Bottom NavigationにもSOSが常時表示される。
  - SOS初期画面に入力必須項目がない。
  - 10秒の停止、60秒の身体行動、3分の気分転換が提示される。
  - SOS画面では短い文言と少ないボタンを使い、押した瞬間に次の行動が分かる。
  - 完了後は `ホームへ`, `あとで振り返る`, `戻った日を記録` の3出口に絞る。

## US-04 記録が途切れたユーザー

- ユーザーストーリー: 記録が途切れたユーザーとして、私は責められずに再開したい。なぜなら、罪悪感が強いとアプリを消したくなるから。
- 背景課題: `失敗` や `リセット` 中心の体験は自己否定を強め、再開を妨げる。
- 体験イメージ: relapseを記録しても `失敗しました` と表示せず、続いた事実と次の一歩を示す。
- 対応画面: Relapse Log, Restart Flow, Home, Calendar, Insights
- MVP: 入れる
- 優先度: P0 Critical
- 受け入れ条件:
  - `失敗`, `我慢できなかった`, `リセット` を主要UIに表示しない。
  - 最長記録、累計成功時間、再開回数が残る。
  - relapse原因はタグ中心で簡単に記録でき、メモは任意。
  - time + next 24h action だけでもRestart Flowを完了できる。
  - 完了後Homeは `次の24時間` に集中した表示になる。

## US-05 人に見られたくないユーザー

- ユーザーストーリー: プライバシーを重視するユーザーとして、私はアプリ名、通知、画面表示を人に見られても恥ずかしくない形にしたい。なぜなら、禁欲や衝動の記録は非常にセンシティブだから。
- 背景課題: ロック画面通知、アプリ切替画面、画面タイトル、エクスポートファイルから目的が露出するリスクがある。
- 体験イメージ: 通知は `今日のチェックイン`, `ひと息つく時間です` など中立的に表示され、アプリロックとデータ管理を設定できる。
- 対応画面: Splash, Home, Settings, Privacy Lock, Data Export/Delete, Notification Settings
- MVP: 入れる
- 優先度: P0 Critical
- 受け入れ条件:
  - 通知に `禁欲`, `ポルノ`, `衝動`, `再発` など直接表現が出ない。
  - アプリ名、画面タイトル、アイコン方針が中立的である。
  - アプリロックを用意する。
  - データ削除とエクスポートを設定画面に用意する。
  - ローカル保存方針とエクスポート後の保護外リスクを説明する。

## US-06 目標を忘れがちなユーザー

- ユーザーストーリー: 継続の意味を忘れがちなユーザーとして、私は `なぜ続けるのか` を定期的に思い出したい。なぜなら、日数だけではモチベーションが続かないから。
- 背景課題: 日数中心の設計は、目的を失うと継続理由も失いやすい。
- 体験イメージ: 初回設定で `集中力`, `自信`, `睡眠`, `時間`, `人間関係`, `勉強`, `仕事の成果` などを選び、HomeやSOSで短く再表示される。
- 対応画面: Goal Setup, Home, SOS, Insights
- MVP: 入れる
- 優先度: P0
- 受け入れ条件:
  - Goal Setupで `取り戻したいもの` を選択できる。
  - Homeに理由カードが表示される。
  - SOS画面に理由が短く再表示される。
  - 日数より生活価値を中心にしたコピーになっている。

## US-07 将来ブロッカーを使いたいユーザー

- ユーザーストーリー: 自力だけでは不安なユーザーとして、私は将来的にブロッカー機能で誘惑を減らしたい。なぜなら、衝動が来た瞬間の意思の力だけでは限界があるから。
- 背景課題: 初回から重い権限を求めると不信感が生まれる一方、将来の強化余地は示しておきたい。
- 体験イメージ: MVPでは実装しないが、SettingsやSOSから `将来のブロッカー` の説明へ自然につながる。
- 対応画面: Settings, Future Blocker Placeholder, SOS, Home
- MVP: プレースホルダーのみ入れる。ブロッカー実装は外す。
- 優先度: P1
- 受け入れ条件:
  - MVPではAccessibility/VPN/UsageStatsなどを要求しない。
  - 初回オンボーディングでブロッカー権限を要求しない。
  - 将来実装時の取得するもの/しないもの/解除方法/誤検知時UXが文書化されている。
  - Future Blocker Placeholderに `必要になった時だけ説明して設定します` と表示する。

## MVP In / Out

MVPに入れる:

- US-01 初めて使う
- US-02 毎日続ける
- US-03 SOS
- US-04 記録が途切れた後の再開
- US-05 プライバシー
- US-06 目標の再表示
- US-07 のプレースホルダー

MVPから外す:

- ブロッカー本体
- 外部/公開コミュニティ
- 課金、広告、評価依頼
- 伴走者通知
- 医療的診断や治療プログラム
