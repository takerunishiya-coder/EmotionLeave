# UX Definition

## Product UX Concept

EmotionLeave は、日本語ネイティブ向けの「衝動コントロールと習慣の立て直し」を支える自己管理アプリである。禁欲そのものを目的化せず、ユーザーが取り戻したい生活、集中、自信、時間に接続する。

UXの合言葉は、**責めない、続け直せる、人に見られても恥ずかしくない**。

## Value Provided

- 衝動が来る前: 朝の誓約と軽い回避プランで、今日の軸を作る。
- 衝動が来た瞬間: SOSから10秒、60秒、3分の短い行動に逃がす。
- 一日が終わる時: 10〜30秒の振り返りで、きっかけと対処を残す。
- 戻ってしまった時: 記録をゼロ扱いせず、最長記録、累計成功時間、再開回数を残す。
- 将来のブロッカーへ: まず信頼を作り、重い権限要求は必要になった時に説明する。

## What We Do Not Do

- 初回起動で課金画面を出さない。
- MVPでは広告を入れない。
- MVPでは公開コミュニティを入れない。
- MVPでは重いブロッカー権限を急がない。
- 医療、治療、診断、完治を標榜しない。
- relapse を人格や意志の弱さとして扱わない。
- 性的指向、嗜好、同意ある成人の性を道徳的に裁かない。

## UX Principles

1. まず信頼を得る  
   初回3分以内で始められ、課金、評価依頼、重い権限要求を後回しにする。

2. 今日だけに分解する  
   90日や一生の目標ではなく、次の24時間、次の10分、次の30秒に落とす。

3. 続いた事実を残す  
   現在記録だけでなく、最長記録、累計成功時間、再開回数、チェックイン回数を肯定的に扱う。

4. 見られても平気な言葉を使う  
   アプリ名、通知、画面見出しは「自己管理」「記録」「整える」に寄せる。

5. 説明できる制限にする  
   将来ブロッカーは「なぜ止めたか」「どう解除するか」「何が記録されるか」を明示する。

6. 入力より行動を優先する  
   高衝動時に長文や反省を求めず、呼吸、移動、メモ、遅延に誘導する。

## Competitive UX Strategy

単なる日数カウンターでは、Quitzilla、KinTimer、禁欲スカイウォーカーの軽さに埋もれる。EmotionLeave は「日本語ネイティブ」「責めない再開」「SOS」「local-first」「将来の説明可能なブロッカー」で勝つ。

競合に勝つ体験方針:

- 初回ペイウォール不信に対し、初回は価値体験のみ。
- 広告割り込みに対し、MVP広告なし。
- 誤検知ストレスに対し、将来ブロッカーは理由タグと一時許可を前提にする。
- relapse後離脱に対し、最長記録と累計成功時間を残す。
- 日本語UXの弱さに対し、翻訳調でなく自然な短文を採用する。
- コミュニティ疲れに対し、MVPでは入れず、将来も安全設計前提にする。

## MVP UX Scope

MVPに入れる:

- Onboarding: 3分以内、目標、理由、初回誓約、通知の任意説明。
- Home: 現在記録、最長記録、累計成功時間、今日の状態、SOS導線。
- Daily Pledge: 朝の誓約、回避プラン、任意メモ。
- Daily Review: 衝動有無、トリガータグ、対処、明日の一手。
- SOS: 10秒/60秒/3分の介入、メモ、あとで振り返る導線。
- Relapse Recovery: 責めない記録、原因タグ、次の24時間、再誓約。
- Calendar/Insights: MVPでは軽量。日別状態、トリガー傾向、累計成功時間を表示。
- Settings: 通知、アプリロック、データエクスポート/削除、将来ブロッカー入口。

MVPから外す:

- 公開コミュニティ
- 伴走者通知
- AI画像判定
- VPN/Device Admin型ブロッカー
- 課金、広告
- 医療的アセスメント

- 対象年齢を 18+ 推奨にするか、ストア上で明確に18+へ寄せるか。

- 「リラプス」という語を主要UIで使うか、「戻った日」と併記するか。
