# Relapse Recovery Flow

## 原則

- relapseを「失敗」ではなく、起きたイベントとして扱う。
- 記録をゼロにするだけにしない。
- 最長記録、累計成功時間、再開回数を肯定的に表示する。
- 次の24時間に集中する。
- 詳細説明や反省文を強制しない。

## Flow

1. 記録するか選ぶ
   - `起きたことを、責めずに記録します`
   - CTA: `記録する`, `今はスキップ`

2. 時刻
   - `今`, `今日`, `昨日`, `日時を選ぶ`

3. 気持ち
   - `悔しい`, `疲れた`, `焦っている`, `ほっとした`, `よく分からない`

4. きっかけ
   - `夜更かし`, `SNS`, `一人の時間`, `ストレス`, `退屈`, `飲酒`, `疲れ`, `その他`

5. 任意メモ
   - placeholder: `何が重なって起きたかを書く`
   - スキップ可

6. 次の24時間の一手
   - `寝る前にスマホを別の場所へ`
   - `通知時間を変える`
   - `SOSをホーム上部に固定`
   - `将来ブロッカー設定を見る`

7. 再誓約
   - `次の24時間を一緒に整えましょう`

## Positive Progress Display

Relapse後の完了画面:

- `最長記録: 18日`
- `累計成功時間: 432時間`
- `再開回数: 4回`
- `新しい記録: 今日から`

表示文:

`ここまで続けたことも、今日から始めることも、どちらも事実です。`

## Copy Rules

使う:
- `起きたこと`
- `戻った日`
- `再スタート`
- `次の24時間`
- `材料`

避ける:
- `失敗`
- `我慢できなかった`
- `リセット`
- `負けた`
- `またやった`

## Edge Cases

- 記録だけして再誓約しない: Homeに `再誓約できます` を表示。
- relapse時刻が開始日より前: エラーではなく `開始日を確認してください`。
- 危機語彙がメモにある: 通常の励ましではなく相談先/緊急ヘルプ導線を出す。
- 連続記録の計算: 現行記録は更新、最長記録と累計成功時間は保持。
