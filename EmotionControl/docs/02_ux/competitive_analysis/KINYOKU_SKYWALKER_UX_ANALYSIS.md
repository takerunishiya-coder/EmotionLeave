## 競合の強み

### 1. 目的がすぐ分かるホーム画面

ホーム画面で、現在の継続日数、時間、目標達成までの日数が即座に分かる。禁欲アプリとしての中心価値が明確で、迷わず使える。

EmotionLeave への転用方針：
- ホームでは「現在の継続」「今日の一手」「なぜ続けるか」をすぐ見せる。
- 単なる日数だけでなく、ユーザーが取り戻したい生活・集中・時間・自信につなげる。

### 2. 機能導線がホームに集約されている

リセット、追加、目標設定、使い方、チャット、シェアがホームから直接押せる。初心者でも主要機能にアクセスしやすい。

EmotionLeave への転用方針：
- ホームに主要導線を置く設計は参考にする。
- ただし、EmotionLeaveでは「SOS」「今日の振り返り」「理由の再表示」を優先し、ボタン数を絞る。

### 3. 目標日数の設定が簡単

目標日数を数字で入力して決定するだけなので、心理的負荷が低い。

EmotionLeave への転用方針：
- 初回設定は3分以内で完了させる。
- 目標日数だけでなく、「なぜ続けるか」「衝動が来たら何をするか」も軽く設定できるようにする。

### 4. カレンダーとメモで振り返れる

成功日・失敗日を色で分け、メモを残せる。継続状況を後から振り返る導線がある。

EmotionLeave への転用方針：
- カレンダーは採用する価値が高い。
- ただし「成功／失敗」の二分法だけでなく、「続けられた日」「立て直した日」「衝動が強かった日」など肯定的な分類にする。

### 5. 称号・目標によるゲーミフィケーション

称号、次の称号までの残り日数、獲得称号があり、継続のモチベーションになる。

EmotionLeave への転用方針：
- 称号やバッジは有効。
- ただし「偉くなる」よりも、「自分を整えた実績」「再開できた力」を可視化する方向に寄せる。

### 6. パスワードロックがある

禁欲アプリは人に見られたくないため、パスワードロックは強い機能。

EmotionLeave への転用方針：
- 初期MVPでもアプリロック・通知文言・表示名のプライバシー配慮は重要。
- 生体認証、PIN、通知の中立表現を検討する。

### 7. 外部コミュニティへの導線がある

LINEオープンチャットへの導線があり、孤独感を減らす方向性は良い。

EmotionLeave への転用方針：
- 伴走・コミュニティの価値は高い。
- ただしMVPでは外部コミュニティ誘導を主軸にしない。将来実装する場合は、トリガー表現、未成年配慮、通報・モデレーション方針を整理する。

## 改善点・弱み

### 1. 広告が目立つ

多くの画面上部に広告があり、センシティブなアプリとしては信頼感を損ないやすい。集中したいタイミングで広告が見えると、体験が軽く見える。

EmotionLeave の強みに変える方針：
- MVPでは広告を入れない。
- 課金する場合も、初回ペイウォールではなく、信頼形成後に自然に提示する。
- 「広告なし・ローカル優先・プライバシー配慮」を訴求価値にできる。

### 2. ホームのボタンが多く、優先順位が分かりにくい

主要導線が多く並び、何を最初に押せばよいか分かりづらい。衝動が来た瞬間に迷う可能性がある。

EmotionLeave の強みに変える方針：
- ホームの主CTAは「SOS」「今日のチェックイン」「理由を見る」の3つ程度に絞る。
- リセットや設定は主導線から少し下げる。
- 衝動時は1タップでSOS画面に入れるようにする。

### 3. relapse時の心理設計が弱い

リセットボタンはあるが、再開体験・振り返り・自己否定を避ける文言が見えない。「リセット＝全部ゼロ」の印象が強くなりやすい。

EmotionLeave の強みに変える方針：
- 「リセット」ではなく「記録を更新」「立て直す」などの表現を検討する。
- 最長記録、累計成功時間、再開回数を保持する。
- relapse後は「次の一手」を提示する。

### 4. 目標設定が数値だけで終わっている

目標日数の設定は簡単だが、なぜその目標を立てるのか、達成後にどうなりたいのかが弱い。

EmotionLeave の強みに変える方針：
- 目標日数に加えて「取り戻したいもの」を選ばせる。
- 例：集中、睡眠、時間、自信、人間関係、仕事・勉強。
- ホームやSOS画面で理由を再表示する。

### 5. カレンダーが成功・失敗の二分法に寄っている

成功日と失敗日が色分けされるが、失敗日が強く可視化されると、ユーザーによっては自己否定や離脱につながる。

EmotionLeave の強みに変える方針：
- 「立て直し日」「記録した日」「衝動をやり過ごした日」などのポジティブ分類を導入する。
- 失敗の可視化より、学習と再開の可視化を重視する。

### 6. LINEオープンチャット導線は便利だがリスクがある

外部コミュニティは励みになるが、発言内容がトリガーになる可能性、比較疲れ、未成年配慮、モデレーション問題がある。

EmotionLeave の強みに変える方針：
- コミュニティはMVPから外す。
- 将来は「匿名・安全・ルール明確・通報あり」の設計にする。
- まずは個人内で完結する伴走UXを強くする。

### 7. 設定画面のプライバシー説明が不足している

パスワードロックはあるが、データ保存、削除、エクスポート、通知文言、権限説明などの安心材料が少ない。

EmotionLeave の強みに変える方針：
- 設定画面に「データ管理」「通知の表示名」「アプリロック」「エクスポート」「削除」を置く。
- ユーザーが安心して使える説明を入れる。

### 8. 「禁欲」感が強く、人に見られると恥ずかしい可能性がある

画面内に禁欲という言葉が明確に出るため、人前で開きづらいユーザーがいる可能性がある。

EmotionLeave の強みに変える方針：
- アプリ名、通知、画面タイトルは中立表現に寄せる。
- 例：「チェックイン」「今日の記録」「集中を整える」「立て直しメモ」。
- 必要な場所では明確に説明しつつ、日常画面では目立たせすぎない。

## EmotionLeaveで強みにすべきUX仮説

1. 競合よりもプライバシーに強い
   - 広告なし、ローカル優先、アプリロック、通知文言の中立化。

2. 競合よりもrelapse後に戻りやすい
   - 記録をゼロにするだけでなく、累計実績・最長記録・再開回数を残す。

3. 競合よりも衝動時の導線が速い
   - 1タップでSOS画面へ遷移し、10秒・60秒・3分の回避アクションを出す。

4. 競合よりも「なぜ続けるか」が見える
   - 目標日数だけでなく、取り戻したい生活・集中・自信・時間をホームに表示する。

5. 競合よりも日本語が自然で責めない
   - 説教、根性論、強すぎる表現を避ける。

6. 競合よりもブロッカー導入が納得できる
   - 将来ブロッカーを入れる場合は、誤検知説明、解除理由、クールダウン、権限説明を設計する。

## Codexへの推奨指示

この画像群は、EmotionLeave の競合UX分析資料として使用してください。

やること：
- 画像を `docs/02_ux/competitive_screenshots/kinyoku_skywalker/` に配置する。
- このメモを `docs/02_ux/competitive_analysis/KINYOKU_SKYWALKER_UX_ANALYSIS.md` に配置する。
- 競合の画面をそのまま模倣せず、強み・弱みを抽象化して `UX_DEFINITION.md`、`COMPETITOR_PAIN_TO_UX_STRATEGY.md`、`SCREEN_FLOW.md`、`SCREEN_SPEC.md` に反映する。

やらないこと：
- 画面デザイン、配色、広告配置、文言、称号名、アイコン配置をそのままコピーしない。
- 競合アプリ名やスクリーンショットをアプリ内に表示しない。
- 医療効果や依存症治療を断定しない。
