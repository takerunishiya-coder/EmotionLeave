# Competitor Pain To UX Strategy

## Strategy Summary

競合市場は「強いが怖いブロッカー」と「優しいが介入できないカウンター」に分かれている。EmotionLeave は、MVPでは軽く信頼を作り、将来は説明可能なブロッカーへ自然につなげる。

## Kinyoku Skywalker Screen Analysis

参照資料:

- `docs/02_ux/competitive_analysis/KINYOKU_SKYWALKER_UX_ANALYSIS.md`
- `docs/02_ux/competitive_screenshots/kinyoku_skywalker/`

抽象化した強み:

- ホームで継続日数、時間、目標までの残りが即座に分かる。
- 主要機能がホームとBottom Navigationに集まり、迷わず触れる。
- 目標日数設定が数値入力だけで軽い。
- カレンダーとメモにより、後から振り返れる。
- 称号/目標進捗が継続動機になる。
- パスワードロックがあり、センシティブアプリとして最低限の安心を作っている。
- 外部チャット導線が孤独感を減らす可能性を持つ。

EmotionLeaveでの差別化要件:

- 広告なしをMVP原則にする。集中したい画面、SOS、記録入力中に広告を出さない。
- ホーム導線は多機能一覧ではなく、`SOS`, `今日のチェックイン`, `理由を見る` を最優先にする。
- `リセット` 中心ではなく、`戻った日を記録`, `次の24時間`, `再誓約` に変換する。
- カレンダーの分類は `成功/失敗` ではなく、`整った日`, `揺れた日`, `立て直した日` にする。
- 称号は動物名や階級名を模倣せず、行動ベースのバッジとして設計する。
- LINE等の外部コミュニティ誘導はMVP外。将来は匿名性、トリガー回避、通報、年齢配慮を前提にする。
- パスワードロックに加えて、生体認証/PIN、通知の中立化、エクスポート/削除説明まで含める。

## Competitive Differentiation Hypotheses

1. 「最初に売らない」ことで、初日離脱と不信レビューを減らせる。
2. 「relapseしても記録が残る」ことで、ストリーク崇拝による離脱を減らせる。
3. 「SOSが1タップ」なら、ブロッカーなしMVPでも衝動瞬間の価値を出せる。
4. 「日本語で恥ずかしくない通知」なら、国内ユーザーの日常利用に入りやすい。
5. 「将来ブロッカーは説明可能」と先に設計しておくことで、強制力追加時の不信を抑えられる。

## MVP Priority

P0:
- 3分オンボーディング
- Homeで現行/最長/累計を併記
- Daily Pledge/Review
- SOS
- Relapse Recovery
- Privacy/Data管理
- 日本語マイクロコピー

P1:
- Calendar/Insightsの軽量分析
- 将来ブロッカー入口
- ブロック理由画面の仕様

P2:
- 伴走者
- コミュニティ
- AIブロッカー
- 課金

## Risks

- ブロッカーをMVPに含めるかは既存文書間で揺れがある。今回のUX定義では「MVPはプレースホルダー、実装は後続判断」とする。
- 「relapse」「リカバリー」は治療文脈に近く見えるため、UIでは「戻った日」「再スタート」「次の24時間」を併用する。
