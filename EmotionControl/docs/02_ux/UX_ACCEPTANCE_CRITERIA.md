# UX Acceptance Criteria

## 共通UX原則

- [ ] 医療効果を断定しない。
- [ ] 説教、断罪、羞恥を誘う文言を使わない。
- [ ] 主要操作は日本語ネイティブ品質で、意味が1回で分かる。
- [ ] オフラインでも pledge / review / relapse / SOS / delete が破綻しない。
- [ ] 権限拒否時も、記録、振り返り、relapse recovery は利用できる。
- [ ] 機微情報は最小表示、最小保存、ログマスクされる。
- [ ] 各画面から `戻る`, `後で設定`, `ホームへ` の逃げ道が明確。

## ユーザーストーリー別受け入れ条件

### US-01 初めて使う
- [ ] 初回設定は3分以内でHomeへ到達できる。
- [ ] 初回に課金、広告、評価依頼、アカウント作成、重い権限要求が出ない。
- [ ] 初回オンボーディング中に強い課金画面、無料トライアル誘導、プラン比較が表示されない。
- [ ] 開始日、目標、取り戻したいものを登録できる。
- [ ] `生活を整えるため` の説明になっており、医療・治療・診断の表現がない。
- [ ] 詳細な性的内容、実名、医療歴を聞かない。

### US-02 毎日続けたい
- [ ] Daily Pledge/Reviewのquick pathは10〜30秒で完了できる。
- [ ] standard pathは1分以内、detail pathは任意で2分以内。
- [ ] Homeに現行記録、最長記録、累計成功時間、再開回数、今日のひとことが表示される。
- [ ] 現行記録だけが突出して大きい設計になっていない。
- [ ] missed check-inでも記録は途切れず、Homeに `今からでも大丈夫です` を表示する。

### US-03 衝動が来た
- [ ] Homeの第一表示範囲からSOS画面へ1タップで遷移できる。
- [ ] Bottom NavigationにもSOSが常時表示される。
- [ ] SOS画面は入力必須なしで開始できる。
- [ ] SOSは10秒自動停止、30秒呼吸、60秒身体行動、3分行動を提示する。
- [ ] SOS完了後の出口は `ホームへ戻る`, `あとで振り返る`, `戻った日を記録する` の3つに絞られる。
- [ ] Freeユーザーでも基本SOSを利用できる。
- [ ] SOS画面に広告が表示されない。
- [ ] SOS中にPlus課金誘導が表示されない。
- [ ] AI SOSはPlus機能として扱われる。

### US-04 記録が途切れた
- [ ] relapse記録後に `失敗` という文言が表示されない。
- [ ] `我慢できなかった`, `リセット` を主要UIに表示しない。
- [ ] 最長記録、累計成功時間、再開回数が保持される。
- [ ] time + next 24h actionだけでもRestart Flowを完了できる。
- [ ] 完了後Homeは `次の24時間` に集中した表示になる。
- [ ] relapse記録直後に課金誘導、広告視聴要求、Plus比較が表示されない。

### US-05 人に見られたくない
- [ ] 通知に `禁欲`, `ポルノ`, `衝動`, `再発`, `リラプス` が出ない。
- [ ] Splash、通知、画面タイトル、ストア向け説明は中立表現を優先する。
- [ ] アプリロックを用意する。
- [ ] Pledge, Review, SOS memo, Relapse Log, Calendar detail, Export/Deleteはアプリ切替画面ぼかし/secure flag対象。
- [ ] データ削除とエクスポートがSettingsから利用できる。
- [ ] データ削除、エクスポート、プライバシー設定はFreeでも利用できる。
- [ ] ローカル保存方針とエクスポート後の保護外リスクを説明する。

### US-06 目標を忘れがち
- [ ] Goal Setupで `集中力`, `自信`, `睡眠`, `時間`, `人間関係`, `勉強`, `仕事の成果` などを選べる。
- [ ] Homeに理由カードが表示される。
- [ ] SOS画面に理由が短く再表示される。
- [ ] 日数より生活価値を中心にしたコピーになっている。

### US-07 将来ブロッカー
- [ ] MVPではブロッカー本体を実装しない。
- [ ] 初回オンボーディングでAccessibility/VPN/UsageStats/Device Adminを要求しない。
- [ ] Future Blocker PlaceholderはSettings/SOSから参照できる。
- [ ] 将来実装時の取得するもの/しないもの、解除方法、一時停止、誤検知時UXが文書化されている。
- [ ] 権限要求前に目的、取得範囲、端末内処理、OFF時の影響を説明する。

## 実装前チェックリスト

- [ ] PRD、UX flow、Privacy、Data model と対象画面の責務が一致している。
- [ ] MVP対象か Phase 2以降かを画面仕様に明記している。
- [ ] 必要な権限、拒否時の代替体験、再設定導線を定義している。
- [ ] コピーは「共感、具体、選べる行動」になっている。
- [ ] データ保存項目、削除対象、エクスポート対象が定義済み。
- [ ] 画面遷移図に成功、キャンセル、エラー、権限拒否ルートがある。
- [ ] legal-policy と privacy-security のレビュー観点が反映されている。

## Monetization / Ads / AI Plus 受け入れ条件

### 広告関連

- [ ] Freeユーザーには対象画面でバナー広告が表示される。
- [ ] Plusユーザーには広告が表示されない。
- [ ] SOS画面に広告が表示されない。
- [ ] レベルアップ演出中に広告が表示されない。
- [ ] 広告が入力欄や主要CTAを妨げない。
- [ ] 広告SDKにメモ本文、relapse詳細、SOS履歴、トリガー名が渡らない。
- [ ] 広告はrelapse履歴、衝動、性的行動、健康状態に近い自己申告をもとにパーソナライズされない。

### SOS関連

- [ ] Freeユーザーでも基本SOSを利用できる。
- [ ] SOS中にPlus課金誘導が表示されない。
- [ ] AI SOSはPlus機能として扱われる。
- [ ] SOS完了後のAI振り返り案内は、落ち着いたタイミングの任意導線である。
- [ ] `Plusなら防げます` のような恐怖訴求が表示されない。

### AI分析関連

- [ ] AI分析前に同意画面が表示される。
- [ ] AI分析で医療的診断表現が使われない。
- [ ] ユーザーがAI分析用データを削除できる。
- [ ] メモをAI分析に含めるか選択できる。
- [ ] relapse記録をAI分析に含めるか選択できる。
- [ ] AI分析を使わなくても基本カウンター、Daily Pledge、Daily Review、基本SOS、relapse記録、カレンダーを利用できる。
- [ ] AI提案は参考情報として表示され、閉じることができる。
- [ ] AIプロバイダーへ送信するデータ、保持、学習利用の有無が説明される。

### 課金関連

- [ ] 初回オンボーディング中に強い課金画面が表示されない。
- [ ] relapse記録直後に課金誘導が表示されない。
- [ ] Plus案内は詳細分析やAI機能を開いたタイミングに限定される。
- [ ] Plusを断っても元のタスクに戻れて、入力内容が失われない。
- [ ] 有料画面に価格、請求周期、自動更新、無料トライアル終了後の扱い、解約方法、Freeで残る機能が表示される。
- [ ] Restore purchase、購入失敗、オフライン時の購入状態、解約/管理導線のテスト観点がある。

## 実装後チェックリスト

- [ ] 初回オンボーディングは3分以内にHome到達できる。
- [ ] 朝の pledge は1分以内、夜の review は2分以内に完了できる。
- [ ] SOSはHomeから1タップで開ける。
- [ ] relapse後も最長記録と現行記録が正しく併記される。
- [ ] データ全削除後、Home、履歴、DB、エクスポートに復元残りがない。
- [ ] JSON/CSVエクスポート内容がUI表示と一致する。
- [ ] 通知文言がロック画面で目的を露出しない。
- [ ] 主要E2Eで重大クラッシュがない。

## 手動テスト観点

### Onboarding
- [ ] Splash -> Onboarding -> Goal Setup -> First Pledge -> Homeへ進める。
- [ ] 権限を許可しなくてもHomeへ進める。
- [ ] 初回で課金、広告、評価要求が出ない。
- [ ] ブロッカー説明が「監視」ではなく「自己管理補助」と伝わる。
- [ ] 戻る操作で入力内容が不自然に消えない。

### Daily
- [ ] 朝の誓約を固定文のみで保存できる。
- [ ] 任意メモ、気分、回避プランを保存できる。
- [ ] 夜の振り返りで衝動有無、トリガー、対処、明日の1アクションを記録できる。
- [ ] 同日重複時の更新、上書き、再編集ルールが明確。
- [ ] 連続日数、累計成功時間が日付変更後も正しい。

### SOS
- [ ] Homeまたは将来ブロック画面から即時起動できる。
- [ ] 10秒自動停止、30秒呼吸、場所移動、水を飲む、メモの導線がある。
- [ ] 完了後にHome、Review、Relapseのいずれかへ戻れる。
- [ ] 危機語彙入力時はヘルプ導線を表示する。
- [ ] SOS使用履歴が必要以上に露出しない。
- [ ] SOS中に広告、課金誘導、評価要求が出ない。

### Relapse
- [ ] relapse記録は任意入力で開始できる。
- [ ] 感情チェック、トリガー選択、次の一手、再誓約へ進める。
- [ ] 文言が責めない表現になっている。
- [ ] 現行記録は更新され、最長記録は保持される。
- [ ] relapse後のHomeが `次の24時間` に集中した表示になる。

### Privacy / Permissions
- [ ] Notification、Privacy Lock、将来Accessibility/VPN等の目的が個別に説明される。
- [ ] 権限オフ時の影響と再設定導線が表示される。
- [ ] 権限拒否後も記録系機能が使える。
- [ ] 収集データ、保存場所、目的外利用なしが確認できる。
- [ ] 医療、法的に未確定な項目は断定しない。
- [ ] AI分析同意、メモ分析のON/OFF、relapse記録分析のON/OFF、AI分析用データ削除がSettingsから確認できる。
- [ ] App Store / Google Play公開前に、実装SDKと実際のデータフローがPrivacy Nutrition Label / Data Safety / プライバシーポリシーと一致している。
- [ ] クラッシュログ、分析イベント、広告SDKにメモ本文、relapse詳細、トリガー名、ブロッカー詳細が含まれない。
- [ ] アカウントを導入する場合、アプリ内からアカウント削除を開始できる。
- [ ] 機微DBをAndroid Auto Backup / iCloud backupの対象にするか除外するかが仕様として決まっている。

### Data
- [ ] pledge / review / relapse / SOS memo / settings が保存される。
- [ ] エクスポートはJSON/CSVで生成できる。
- [ ] エクスポート前に含まれるデータ範囲が分かる。
- [ ] 全削除は確認ステップ後に即時反映される。
- [ ] 全削除後に履歴、統計、監査ログ、設定が残らない。
- [ ] AI分析用の要約、特徴量、キャッシュ、プロンプト、レスポンスが削除対象に含まれる。
- [ ] 端末内記録は暗号化ローカルDBに保存され、鍵はOSの安全な鍵管理を使う。
- [ ] エクスポートファイルはアプリ保護外になることが生成前に説明される。

### Monetization / AI Plus
- [ ] Paywall / Settings / Store説明 / About / Empty state / Error / Notificationの日本語文言に禁止表現がない。
- [ ] Plus案内は `高度な振り返り`、`記録から見える傾向`、`次に試せる工夫` として表現される。
- [ ] 購入復元、購入失敗、サブスクリプション管理、オフライン時の状態表示を確認する。
- [ ] Plus解約後もFree機能、データ削除、エクスポート、基本SOSが利用できる。

### Future Blocker
- [ ] MVPは将来機能として表示し、画像AI・VPN方式は後回しとして明記する。
- [ ] ブロック理由タグはURLルール、キーワード一致、権限停止などを区別する仕様になっている。
- [ ] 解除は今回のみ / 15分 / 24時間から選べ、理由入力が必要。
- [ ] 解除結果は監査ログに残る。
- [ ] ブロッカー停止・権限失効時は自己診断と復旧導線を出す。
- [ ] 端末差・省電力制限で動作しない可能性を説明できる。

## 画面遷移テストポイント

- [ ] 初回起動 -> オンボーディング完了 -> Home
- [ ] 初回起動 -> 権限拒否 -> Home -> 後から権限設定
- [ ] Home -> Daily Pledge -> 保存 -> Home統計更新
- [ ] Home -> Daily Review -> 保存 -> Home統計更新
- [ ] Home -> SOS -> アクション完了 -> Home
- [ ] Home -> Relapse Log -> Restart Flow -> Re-Pledge -> Home
- [ ] Settings -> Privacy Lock -> 有効化 -> 再起動 -> ロック解除
- [ ] Settings -> Export -> JSON/CSV確認 -> Settings
- [ ] Settings -> Delete -> 初期状態HomeまたはOnboarding
- [ ] Future Blocker Placeholder -> Permission Info -> Settings

## UX上の未決事項

- MVPでブロッカーを完全に除外するか、URL/キーワードのみPhase 1に含めるか。
- アプリ内で `リラプス` をどの程度使うか。
- 年齢制限をストア上で18+へ寄せるか。
- 匿名テレメトリを導入するか、完全local-onlyで始めるか。
