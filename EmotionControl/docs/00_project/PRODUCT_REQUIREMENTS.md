# StopEmotion - PRODUCT REQUIREMENTS (MVP中心)

## 0. プロダクト定義
StopEmotionは「依存症の治療アプリ」ではなく、**自己管理・習慣改善を支援する Explainable Recovery Copilot**。

中核価値:
1. 継続（daily pledge / review）
2. 抑止（段階的 blocker）
3. 納得（なぜ止めたか説明可能）

## 1. 解くべき課題（調査反映）
- 初回ペイウォール不信。
- 誤検知・誤ブロック時に理由不明。
- ブロッカーの不安定性。
- 解除フローが厳しすぎる/分かりづらい。
- 広告過多と日本語UX品質不足。
- データ削除/エクスポート弱さ。

## 2. 対象ユーザー
- 主: 日本語ネイティブのAndroidユーザー（18歳以上推奨）
- 副: iOS将来対応を見据える自己管理ニーズ層

## 3. MVPスコープ（Must）
### 3.1 デイリーループ
- 朝: daily pledge（1タップ + 自由記述任意）
- 夜: daily review（衝動有無、トリガー、対処）
- 連続日数、累積時間の可視化

### 3.2 リカバリー支援
- リラプス記録（責めない文面）
- 「最長記録」と「現行記録」を併記
- urge helper（呼吸/短い行動提案/メモ）

### 3.3 Explainable Blocker (Android phase1)
- ブロック対象: URL/キーワード（まずはブラウザ中心）
- ブロック時に理由タグ表示（例: URLルール、キーワード一致）
- 解除方式: 今回のみ / 15分 / 24時間（要理由入力）
- 監査ログ（いつ・何を・なぜ解除したか）

### 3.4 プライバシー
- ローカル保存優先
- データ削除（全削除）
- データエクスポート（JSON/CSV）

## 4. MVP外（Won't for MVP）
- 課金・広告導入
- 公開コミュニティ機能
- 画像AI誤検知対策付き全画面検知
- VPN型の強制ブロッカー
- 伴走者通知（phase2候補）

## 5. 非機能要件
- オフライン利用可能（主要機能）
- 起動2秒以内（中央値目標）
- クラッシュフリー率 99.5%以上（内部テスト目標）
- 日本語コピーをネイティブ品質で統一

## 6. 安全・倫理要件
- 医療効果を断定しない。
- 危機語彙（自己危害）にはヘルプ導線表示。
- センシティブ情報は最小収集・最小保持。

## 7. 成功指標（MVP）
- D1継続率 > 45%
- D7継続率 > 25%
- 7日以内にdaily pledge 3回以上実施率 > 60%
- ブロック理由表示画面からの離脱率低下（比較実験対象）
