# AI Data Usage Policy

## 基本方針

AI分析は、ユーザーの記録を振り返るためのPlus機能である。医療的な診断や治療ではなく、入力された記録から傾向を整理し、次に試せる工夫を提案するために利用する。

AI分析を使う前に、ユーザーの明示的な同意を取得する。

初期状態ではAI分析はオフにする。AIを使わない場合でも、基本カウンター、Daily Pledge、Daily Review、基本SOS、relapse記録、カレンダー、データ削除、エクスポートは利用できる。

## AI分析に使うデータ

ユーザーがAI分析を有効にした場合、以下のデータを分析対象にできる。

- 継続日数。
- relapse日時。
- relapse前の気分。
- relapseのトリガー。
- SOS使用履歴。
- Daily Pledge。
- Daily Review。
- 睡眠、疲労、ストレスの自己申告。
- メモ。ただしユーザーが許可した場合のみ。

## AI分析に使わないデータ

- 実名や連絡先。
- 端末内の他アプリの内容。
- 位置情報。
- 写真、動画、連絡先。
- 医療歴。
- 決済情報の詳細。
- ユーザーがAI分析対象から除外したメモやrelapse記録。
- ブロッカーで検知したURLやアプリ名の詳細。ただし将来、ユーザーが明示的に同意した場合を除く。

将来ブロッカー機能を追加する場合でも、AI分析に送るデータは別途同意を取り、必要最小限にする。

## ユーザー同意

AI分析利用前に、以下を説明する。

- AI分析では、入力した記録、振り返り、メモをもとに傾向を整理する。
- この機能は医療的な診断や治療ではない。
- 分析を利用するかどうかはいつでも変更できる。
- メモ内容をAI分析に含めるかどうかを選べる。
- relapse記録をAI分析に含めるかどうかを選べる。
- AI分析用データを削除できる。

## ユーザーが選べる設定

- AI分析を使う / 使わない。
- メモ内容をAI分析に含める / 含めない。
- relapse記録をAI分析に含める / 含めない。
- AI分析用データを削除する。

## メモの扱い

メモは特にセンシティブな内容を含む可能性があるため、初期状態ではAI分析に含めない設定を推奨する。ユーザーが明示的に許可した場合のみ、AI分析に含める。

## relapse記録の扱い

relapse記録はAI分析の価値に直結する一方で、センシティブ性が高い。AI分析に含めるかどうかをユーザーが選べるようにする。除外した場合でも、基本カウンター、カレンダー、基本SOSは利用できる。

## AI分析用データ削除

SettingsからAI分析用データを削除できるようにする。削除対象には、AI分析のために作成された要約、特徴量、キャッシュ、履歴を含める。通常のアプリ記録を削除するかどうかは、ユーザーに分かる形で分けて説明する。

## ローカル保存と外部AI送信の違い

ローカル保存は端末内またはアプリ管理領域に保存する状態である。外部AI送信は、分析のために一部データをAIサービスへ送る状態である。ユーザーには、どのデータがローカルに残り、どのデータが外部AI分析に送られる可能性があるかを分けて説明する。

外部AI送信を行う場合は、送信前に同意を取り、送信データを必要最小限にする。ログ、クラッシュレポート、分析イベントにメモ本文やrelapse詳細を含めない。

AIプロバイダーを利用する場合は、利用時点で以下を確認し、アプリ内またはプライバシーポリシーで説明する。

- 送信するデータの種類。
- 送信しないデータの種類。
- プロンプトとレスポンスの保持期間。
- モデル学習に使われるかどうか。
- 不正利用対策やセキュリティ目的で処理されるログやメタデータ。
- サブプロセッサや第三者提供の有無。

## 将来のオンデバイスAI

将来的にオンデバイスAIを検討する余地を残す。オンデバイスAIにより、メモやrelapse記録を外部送信せずに傾向整理できる可能性がある。ただし、MVPでは外部AI利用の有無、送信データ、削除方法を明確にすることを優先する。

## privacy-security review notes

- AI分析は明示的な同意を前提にする。
- メモとrelapse記録は個別に分析対象から外せるようにする。
- AI分析用データ削除はPlus解約後も利用できるようにする。
- 広告SDKにセンシティブな記録内容を渡さない。
- 広告ID、分析ID、クラッシュログとrelapse詳細を結びつけない。
- pledgeメモ、reviewメモ、relapse理由、トリガー、ブロッカーイベント、解除理由、将来の伴走者機能の共有データはセンシティブデータとして扱う。
- センシティブデータは最小保存、ログマスク、目的外利用禁止、保持期間の明確化、削除対象への包含を必須とする。
