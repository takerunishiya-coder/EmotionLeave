# TEST PLAN

## 1. テスト戦略
- Unit: ドメインロジック（連続日数、リラプス計算、ブロック判定）
- UI: Compose画面遷移、文言、入力バリデーション
- Integration: Room保存、DataStore、WorkManager通知
- Manual E2E: オンボード→日次運用→リラプス→復帰

## 2. 重点観点
- 誤ブロック時に理由表示されるか
- 解除フローが分かりやすいか
- 権限拒否でも最低限利用できるか
- データ削除後に復元されないか

## 3. 回帰シナリオ
1) 新規開始
2) 7日継続
3) リラプス記録
4) 解除申請
5) 全削除

## 4. 品質ゲート
- 主要ユースケースの手動E2E合格
- Unit test pass率100%
- 重大クラッシュ0件（検証範囲内）
