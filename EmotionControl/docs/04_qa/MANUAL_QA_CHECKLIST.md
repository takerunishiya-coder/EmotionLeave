# Manual QA Checklist

## Purpose

App Store / Google Play 申請前に、内部テストで最低限確認するユーザーストーリー別チェックリスト。医療的な診断や治療ではなく、記録と振り返りを支援するアプリとして、安心して使える状態かを確認する。

## Test Environment

- Android 12 以上の実機またはエミュレーターで確認する。
- Android 13 以上では通知権限の許可 / 拒否の両方を確認する。
- 新規インストール状態、データ削除後の状態、継続利用状態を分けて確認する。
- スクリーンショットや画面録画は、個人メモが写らないテストデータで取得する。

## Machine Test Gate

- [ ] `testDebugUnitTest` が成功する。
- [ ] `lintDebug` が成功する。
- [ ] `assembleDebug` が成功する。
- [ ] Lint レポートにリリース前ブロッカーがない。
- [ ] 依存関係や SDK バージョンの警告を確認し、申請前リスクとして残すものを明記する。

## User Story 1: 初めて使うユーザー

目的: 初回起動直後に課金圧や不安を感じず、基本価値まで到達できること。

- [ ] 初回起動でオンボーディングが表示される。
- [ ] 初回起動直後に強い課金画面が表示されない。
- [ ] 医療的な診断、治療、依存症断定に見える文言がない。
- [ ] 取り戻したいもの、開始日、基本設定を迷わず入力できる。
- [ ] 入力後にHomeへ到達できる。
- [ ] アプリを再起動してもオンボーディング完了状態が維持される。

## User Story 2: 毎日の記録を残すユーザー

目的: Daily Pledge / Daily Review が短時間で記録でき、Homeの統計に反映されること。

- [ ] Daily Pledge を保存できる。
- [ ] Daily Review を保存できる。
- [ ] メモ未入力でも保存できる。
- [ ] 保存後にHomeへ戻る。
- [ ] Homeに誓約数、振り返り数、今日の状態が実データに基づいて表示される。
- [ ] 入力欄や主要CTAが広告想定領域に隠れない。
- [ ] 危機的な語句を入力した場合、支援先に相談する案内が表示される。

## User Story 3: 強い衝動が来たユーザー

目的: Freeユーザーでも基本SOSをすぐ使え、課金誘導や広告で妨げられないこと。

- [ ] HomeからSOSへ1タップで移動できる。
- [ ] SOS画面に広告表示領域がない。
- [ ] SOS中にPlus課金誘導が表示されない。
- [ ] 10秒 / 短時間アクションを選べる。
- [ ] メモ未入力でもSOSを完了できる。
- [ ] 「あとで振り返る」でHomeへ戻れる。
- [ ] SOS履歴が記録一覧またはHomeの件数に反映される。
- [ ] 危機的な語句を入力した場合、支援先に相談する案内が表示される。

## User Story 4: relapse を記録して再開するユーザー

目的: 責めない再開体験になり、直後に課金誘導されず、継続統計が再開日基準に更新されること。

- [ ] relapse記録画面に責める表現がない。
- [ ] relapse記録直後にPlus課金誘導が表示されない。
- [ ] 記録後にHomeへ戻る。
- [ ] 現行記録が再開日基準で更新される。
- [ ] 最長記録が過去の継続日数を考慮して更新される。
- [ ] relapse回数が増える。
- [ ] 次の一手を未入力にしてもクラッシュしない。

## User Story 5: 通知を使う / 使わないユーザー

目的: 通知が任意で、拒否しても基本機能が使えること。

- [ ] Android 13 以上で通知ON時に権限ダイアログが表示される。
- [ ] 通知権限を拒否してもアプリを継続利用できる。
- [ ] 通知OFFにすると日次リマインダーがキャンセルされる。
- [ ] ワーカー実行時に通知OFF設定なら通知が出ない。
- [ ] 通知タイトルと本文がセンシティブな内容を含まない。
- [ ] 通知からアプリを開ける。

## User Story 6: データを管理したいユーザー

目的: アカウントなしでもローカルデータを管理でき、削除とエクスポートが明確であること。

- [ ] 設定からデータエクスポートを実行できる。
- [ ] 画面内に生のJSONやセンシティブメモが常時表示されない。
- [ ] 共有先選択画面が開く。
- [ ] データ削除前に確認ダイアログが表示される。
- [ ] 全データ削除後、Home / 記録 / 設定の保存済み情報が消える。
- [ ] 削除後に通知がキャンセルされる。
- [ ] 削除後にエクスポートキャッシュが残らない。
- [ ] 削除後にアプリ再起動しても削除済みデータが復元されない。

## User Story 7: 記録を見返すユーザー

目的: 記録一覧が有用で、過度にセンシティブな詳細を不意に露出しないこと。

- [ ] 誓約、振り返り、SOS、relapse記録が時系列で表示される。
- [ ] 空状態が自然に表示される。
- [ ] 長いメモや長いタグでもレイアウトが破綻しない。
- [ ] 画面共有時に過度に詳細なセンシティブ情報が出すぎない。
- [ ] 戻る導線でHomeに戻れる。

## User Story 8: Free / Plus 境界を確認する

目的: Freeの中核価値を壊さず、Plus価値がAI分析に限定されていること。

- [ ] Freeでも基本カウンター、Daily Pledge、Daily Review、基本SOS、relapse記録を使える。
- [ ] AI週次レポート、詳細インサイト、AI SOS提案はPlus機能として扱われる。
- [ ] Plus案内は詳細分析やAI機能を開いたタイミングに限定される。
- [ ] SOS中、relapse直後、初回オンボーディング中に強い課金誘導が出ない。
- [ ] Plus案内に医療効果や治療保証の表現がない。

## Store Review Smoke Check

- [ ] ストア説明文とアプリ内文言で、医療的診断や治療を示唆していない。
- [ ] センシティブデータのローカル保存、削除、エクスポート方針が説明できる。
- [ ] 広告、通知、AI分析、サブスクリプションの有無がストア表示と矛盾しない。
- [ ] スクリーンショットに過度にセンシティブな入力例がない。
- [ ] 審査メモに、AI分析は未実装またはPlus予定機能であることを正確に記載する。

## Release Decision

申請候補にできる条件:

- [ ] 重大クラッシュがない。
- [ ] 初回起動からHome到達まで完走できる。
- [ ] 基本SOSがFreeで使える。
- [ ] relapse直後に課金誘導がない。
- [ ] データ削除が端末上で完了する。
- [ ] 通知拒否、通知OFFでも基本機能が壊れない。
- [ ] ストア文言、プライバシー説明、アプリ内文言に重大な矛盾がない。
