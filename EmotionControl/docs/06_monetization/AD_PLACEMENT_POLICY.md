# Ad Placement Policy

## 基本方針

Freeユーザーには、原則として対象画面でバナー広告を表示する。Plusユーザーには広告を表示しない。広告は収益化のために使うが、衝動時の緊急導線、達成感、入力、主要CTAを妨げない。

広告はセンシティブな記録に基づくパーソナライズをしない。relapse履歴、衝動、性的行動、メモ、ブロッカー履歴、トリガー、健康状態に近い自己申告を広告配信や広告セグメントに使わない。

## 広告を出してよい画面

- ホーム。
- カレンダー。
- インサイト。
- 称号一覧。
- 設定。
- Daily Pledge。
- Daily Review。
- relapse記録。
- Restart Flow。

Daily Pledge、Daily Review、relapse記録、Restart Flowでは広告表示を許容するが、入力欄、保存ボタン、再開導線、戻る導線に被せない。

## 広告を出してはいけない画面・状態

- SOS画面。
- レベルアップ演出中。

## SOS画面に広告を出さない理由

SOS画面は衝動時の緊急導線である。広告が表示されると、行動開始の速度、安心感、信頼感を損なう可能性がある。Freeユーザーであっても、SOS中は広告を表示しない。

## レベルアップ演出中に広告を出さない理由

レベルアップ演出は達成感を強化する場面である。広告が重なると、達成の余韻が損なわれ、継続支援の価値が弱くなる。演出中は広告を非表示にし、演出終了後の通常画面で広告表示を再開する。

## バナー広告中心にする理由

バナー広告はユーザーの操作を遮りにくく、日次利用のリズムを崩しにくい。EmotionLeaveでは、画面下部など操作を邪魔しにくい位置に固定し、主要CTAや入力欄と視覚的に競合しないようにする。

## 全画面広告を避ける理由

全画面広告、動画広告、閉じるまで待たせる広告は、衝動対策アプリの信頼を損ないやすい。特にrelapse記録、Restart Flow、Daily Reviewのような弱さを扱う場面では、待機や強制視聴が搾取的に見えるリスクがある。MVPでは原則使わない。

## 配置ルール

- 広告は画面下部など、操作を邪魔しにくい位置に配置する。
- 入力欄、保存ボタン、SOS導線、再開導線、主要CTAに被せない。
- 画面の最初の入力フォーカスを広告で押し下げない。
- キーボード表示中は広告が入力欄を隠さない。
- 広告の読み込み失敗で主要画面のレイアウトが崩れない。
- Plusユーザーには広告枠そのものを表示しない。
- 広告SDKにメモ本文、relapse詳細、SOS履歴、トリガー名を渡さない。
- 広告IDとセンシティブな記録を結びつけない。

## 実装時の判定

広告表示の可否は、ユーザーのプラン、画面、現在の状態で判定する。

| 条件 | 広告 |
| --- | --- |
| Plusユーザー | 非表示 |
| Freeユーザー + SOS画面 | 非表示 |
| Freeユーザー + レベルアップ演出中 | 非表示 |
| Freeユーザー + 対象画面 | 表示 |

## Store / SDK Review Notes

- 広告SDK導入時は、データ共有、広告ID利用、パーソナライズ有無、Data Safety / Privacy Nutrition Labelの表示内容を更新する。
- 未成年に見えるユーザーや18歳未満を対象にする場合、広告パーソナライズ、性的文脈、年齢ゲートを別途確認する。
- 広告表示はユーザーの弱っている状態を利用した報酬設計にしない。`広告を見ればSOSを続けられます` のような文言は禁止する。
